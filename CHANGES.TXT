New feature in version 2.0

TCC
---

1) The science and guide targets are not contained within a component called
   "field" and each field has its own list of targets.

2) The rotator and guide components have moved from the tcs configuration
   to the field.

3) The airmass limit and Gemini program id have been moved from the science
   target to the field.

4) The observing and guiding wavelengths have been moved from the targets to
   the field and guide components respectively.

4) The targets now has a brightness component; all the TCC does with it is
   to display it so it can be any character string you like.

5) Multiple catalogues can be loaded into the sky plot (aka "radar screen")
   and plotted with a selectable colour. Catalogues can also be unloaded.

6) The guide component can set the peripheral wave-front sensor filters. A 
   default wavelength is provided when the filter is selected.

7) The "preview" window now displays the guide stars' distance from the field
   center in arcmin.

8) Files containing target and other component definitions imported and 
   exported from the TCC are now in XML.

9) The tracking frame component has been removed from the tcs configuration. 
   manager window. They can still be set by opening the tracking frame
   editing window from the Edit menu of the target selector window.

10) OIWFS follow can be controlled from the Aquisition and Guiding screen.

TSD
---

1) The state of the GPOL slides is now displayed on the TSD.

2) The guider effective wavelengths are now displayed on the TSD.

Changes since V2.0a1 released.

1) Target objname defaults to target name.

2) Faster XML parser.

3) Bugs in the interpretation of catalogue epochs fixed.

4) dedication and firstlight catalogues deleted.

5) posAngle added to XML field definition that creates a rotator component with
   cosys set to FK5/J2000.

6) Zero instrument offsets option added to slew. (requires TCS V4-3).

7) "extended" added to the M2 deployable baffle positions.
